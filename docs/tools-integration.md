# Раздел «Инструменты» в workspace

Два самостоятельных инструмента встроены в личный кабинет по принципу
**полулокально**: тяжёлые вычисления идут не на сервере портфолио, а локально /
в CI / в браузере. Сервер только принимает и показывает результат.

Оба доступны в сайдбаре workspace под заголовком **Инструменты**.

## 1. Repo Health (repo-janitor)

Сканирование репозиториев (17 сканеров, git-история) **не** выполняется на
сервере портфолио. repo-janitor гоняется там, где живёт репозиторий (локальный
CLI или GitHub Action) и присылает компактный JSON-отчёт сюда.

### Настройка

1. Выполнить SQL из `docs/workspace-schema.sql` (таблица `ws_tool_reports`).
2. Задать переменную окружения `TOOLS_INGEST_SECRET` (любая случайная строка) —
   и на деплое, и там, откуда шлётся отчёт.
3. Настроить repo-janitor (CLI/Action) слать POST на
   `https://<домен>/api/tools/repo-health/ingest` с заголовком
   `x-tools-secret: <TOOLS_INGEST_SECRET>`.

### Контракт ingest (тело POST, JSON)

```jsonc
{
  "repo": "owner/name",          // обязательно
  "score": 82,                    // 0..100
  "grade": "B",                   // A..F
  "scannedAt": "2026-07-19T...",  // опц., по умолчанию — сейчас
  "summary": { "critical": 1, "warning": 4, "info": 12 },
  "categories": [
    { "name": "Vulnerable deps", "severity": "critical", "count": 1 }
  ],
  "url": "https://.../dashboard"  // опц., ссылка на полный дашборд
}
```

Панель показывает грейд A–F, счёт, тренд по последним сканам и разбивку по
категориям. История хранится (одна строка на каждый присланный отчёт).

## 2. Figma → Code

Порт standalone-приложения figma-to-code как вкладки. Компьют лёгкий и в
основном клиентский; сервер — только тонкий прокси к Figma REST и конвертер.

### Два режима

- **Токен (REST):** вставьте Figma personal access token и ссылку на файл/фрейм.
  Токен хранится в браузере (localStorage). Работает и на хостинге.
  Опционально можно задать `FIGMA_TOKEN` в окружении сервера — тогда поле токена
  можно оставить пустым.
- **Плагин:** companion-плагин Figma шлёт выделение напрямую (без токена и
  лимитов REST). **Только локально:** релей работает через in-memory SSE
  (`/api/plugin/*`), который не переживает serverless (Vercel) — там будет
  молчать. Для локального запуска портфолио — работает.

### Плагин Figma

Исходники в `figma-plugin/` (`manifest.json`, `code.js`, `ui.html`).

1. Figma → Plugins → Development → Import plugin from manifest → выбрать
   `figma-plugin/manifest.json`.
2. В `manifest.json` → `networkAccess.allowedDomains` добавить адрес, куда шлёте
   (по умолчанию `http://localhost:3000`).
3. В окне плагина указать тот же endpoint; выделить блок → «Отправить выбранное».

### Авторизация

Все роуты `/api/figma/*` и `/api/convert` закрыты `requireOwner` (только
владелец). `/api/plugin/*` открыты (плагин шлёт кросс-доменно без сессии) — это
безобидный in-memory релей.
